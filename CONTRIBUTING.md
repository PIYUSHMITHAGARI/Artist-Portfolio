# Contributing to Artist Portfolio

We love your input! We want to make contributing to this portfolio template as easy and transparent as possible.

## Development Process

1. Fork the repo and create your branch from `main`
2. Make your changes
3. Test your changes
4. Submit a pull request

## Pull Request Process

1. Ensure your code follows the existing style
2. Update the README.md if needed
3. Test on multiple browsers and devices
4. Use clear commit messages

## Code Style

### HTML
- Use semantic HTML5 elements
- Include proper accessibility attributes
- Follow consistent indentation (2 spaces)

### CSS
- Use CSS custom properties for theming
- Follow mobile-first responsive design
- Use BEM-like naming conventions

### JavaScript
- Use modern ES6+ features
- Add comments for complex logic
- Handle errors gracefully

## Setting Up Development Environment

1. Clone your fork
```bash
git clone https://github.com/PIYUSHMITHAGARI/Artist-Portfolio.git
cd artist-portfolio
```

2. Test locally by opening `portfolio.html` in a browser

3. Use a local server for full functionality:
```bash
# Python 3
python -m http.server 8000

# Node.js
npx http-server
```

## Areas for Contribution

- **New Features**: Additional portfolio sections, animations, interactive elements
- **Bug Fixes**: Browser compatibility issues, responsive design improvements
- **Documentation**: Better instructions, code comments, examples
- **Performance**: Optimize images, reduce bundle size, improve loading times

## Reporting Bugs

Use GitHub Issues to report bugs. Include:

- Description of the issue
- Steps to reproduce
- Browser and device information
- Screenshots if applicable

## Feature Requests

We welcome feature requests! Please:

1. Check if the feature already exists
2. Explain the problem it solves
3. Suggest implementation ideas
